Parametric Statistical Models

Statistical inference has to reduce complexity: from a rich phenomenon to a tractable representation. A central fork in the road is whether we model the whole distribution flexibly (nonparametric) or commit to a finite-dimensional parameter (parametric).

1) The reduction imperative

Inference becomes possible only after a reductive step: we replace the full complexity of the observed phenomenon by a simplified probabilistic description that can be estimated and compared from finite data. This reduction is not a defect; it is the price of having a coherent calculus of uncertainty.

2) Two approaches: nonparametric vs. parametric

One approach aims to incorporate as much complexity as possible and estimates the underlying distribution under minimal assumptions, often via functional estimation (density estimation, regression functions, etc.). This is the nonparametric stance: the object to be learned is (in effect) infinite-dimensional.

Conversely, the parametric stance represents the distribution through a family \(f(x\mid\theta)\) where the functional form is known and only a finite-dimensional parameter \(\theta\) is unknown. The payoff is pragmatic: finite samples can efficiently learn only finite complexity.

3) Why the parametric choice is pragmatic

Parametric modeling explicitly acknowledges finite-sample limits. If we can only see \(n\) observations, we can reliably estimate only a limited number of degrees of freedom. Moreover, parametric models let us evaluate procedures at finite sample sizes rather than relying solely on asymptotic intuition.

4) The formal object: a parametric statistical model

A compact definition is: a parametric statistical model consists of observing a random variable \(x\) distributed according to \(f(x\mid\theta)\), where only \(\theta\) is unknown and \(\theta\) belongs to a finite-dimensional parameter space \(\Theta\).

In practice we observe a sample \(x_1,\dots,x_n\). A unifying notation is to write the full observation vector \(x = (x_1,\dots,x_n)\) as

\[ x \sim f(x\mid\theta), \]

which covers isolated observations, dependent observations, and i.i.d. sampling. In the i.i.d. case:

\[ f(x\mid\theta)=\prod_{i=1}^{n} f(x_i\mid\theta). \]

5) A Bayesian notation note: why lower-case \(x\)

Some texts reserve capitals \(X\) for random variables and lower-case \(x\) for realizations. In Bayesian work, we almost always condition on the realized data \(x\), and we may treat \(\theta\) itself as random. Keeping \(x\) as “the data” avoids clutter—though we can revert to \(X/x\) when ambiguity matters.

6) What “inference on \(\theta\)” means

Once the statistical model is defined, the aim of analysis is an inference about \(\theta\): we use the observation \(x\) to improve knowledge about \(\theta\), and then support a decision that depends on \(\theta\).

Inference questions vary in form—some target a value (e.g., “what is \(\theta_1\)?”), others a comparison (e.g., “is \(\theta_2\) larger than \(\theta_3\)?”). This is often organized as:

  • Estimation: infer \(\theta\) (or a function \(h(\theta)\)).
  • Testing: assess a hypothesis about \(\theta\).
  • Prediction: infer a future \(y\) whose distribution depends on \(\theta\), e.g. \(y\sim g(y\mid\theta,x)\).

From a decision-theoretic viewpoint, these divisions are partly conventional: estimation and testing can be expressed as special cases of choosing an action to minimize expected loss.
